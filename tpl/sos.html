[[title]] [[sensor_observation_service]]

Ein Sensor Observation Service1 (SOS) dient dem einheitlichen, standardisierten Angebot gemessener Sensordaten. Es handelt sich um einen Standard des Open Geospacial Consortium2 (OGC), welcher zurzeit in Version 2.03 vorliegt. Weitere Informationen zum Standard entnehmen Sie bitte den weiterführenden Links.

Unterstützte Anfragetypen des Aeolus SOS

Unser Sensor Observation Service implementiert das so genannte "Core Profile", welches neben weiteren Profilvarianten wie "Transactional Profile" oder "Enhanced Profile" den Funktionsumfang des Services angibt. Sämtliche Anfragen an den Aeolus SOS laufen über HTTP GET. HTTP POST wird nicht unterstützt. Der Aeolus SOS bietet die folgenden Operationen.

GetObservation

Gibt eine Sensordatenzeitreihe des über den featureOfInterest-Parameter angeforderten Air Quality Eggs in standardisiertem XML Format zurück. Über den observedProperty-Parameter werden (per Komma getrennt) die zurückgegebenen Sensoren spezifiziert. Folgende Parameter sind vom Aeolus SOS implementiert.

Parameter unterstützte Werte Pflichtangabe
request getObservation Ja
service sos Ja
version 2.0.0 Ja
responseFormat text/xml;subtype="om/1.0.0" Ja
observedProperty urn:ogc:def:phenomenon:co
urn:ogc:def:phenomenon:no2
urn:ogc:def:phenomenon:temperature
urn:ogc:def:phenomenon:humidity
(mehrere Werte mit Kommata trennen)		 Ja
offering airqualityegg Ja
featureOfInterest urn:ogc:object:feature:sensor:airqualityegg-[feed-id]
([feed-id] durch cosm Feed ID des Air Quality Eggs ersetzten.)		 Ja
eventTime Startzeit/Endzeit
Zeiten werden im ISO Format angegeben:
YYYY-MM-DDTHH:mm:ss±HH
Beispiel: 2013-04-02T06:01:25+00/2013-04-02T12:01:25+00		 Nein
outlierInterpolation 0,1,2,3 Nein

GetCapabilities

Gibt eine Beschreibung der angebotenen Features des SOS in standardisiertem XML Format zurück. Liefert weiterhin Informationen über die verwendeten Sensoren und die Vorgehensweise bei der Messung. Folgende Parameter sind vom Aeolus SOS implementiert.

Parameter unterstützte Werte Pflichtangabe
request getCapabilities Ja
service sos Ja

DescribeSensor

Gibt Metadaten zum bei der Datenerfassung verwendeten Sensor in standardisiertem XML Format zurück. Folgende Parameter sind vom Aeolus SOS implementiert.

Parameter unterstützte Werte Pflichtangabe
request describeSensor Ja
service sos Ja
version 2.0.0 Nein
outputFormat text/xml;subtype="sensorML/1.0.1" Nein
procedure urn:ogc:object:feature:sensor:airqualityegg-[feed-id]
([feed-id] durch cosm Feed ID des Air Quality Eggs ersetzten.)		 Ja
